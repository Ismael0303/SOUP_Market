# REGLA PARA BACKUPS DE FRONTEND (SOLO CÓDIGO FUENTE)

- Nunca incluir en los backups:
  - node_modules/
  - build/ o dist/
  - .env, .cache, .next, .turbo, coverage, etc.
  - Archivos temporales o de sistema
- Solo respaldar:
  - src/
  - public/
  - package.json, package-lock.json, jsconfig.json, tailwind.config.js, postcss.config.js, etc.
  - README, scripts propios, documentación relevante

## Ejemplo de comando para comprimir solo el código fuente y configuración (Windows CMD):

```
REM Desde la raíz del proyecto frontend
REM Crea un zip solo con lo necesario para restaurar el proyecto
powershell Compress-Archive -Path src,public,package.json,package-lock.json,jsconfig.json,tailwind.config.js,postcss.config.js,README.md -DestinationPath backup_frontend_SOUP_%date:~10,4%-%date:~4,2%-%date:~7,2%.zip
```

## Notas:
- Para restaurar, solo descomprime y ejecuta `npm install` para regenerar node_modules.
- Si usas otro sistema operativo, adapta el comando de compresión (ej: tar, zip, etc.).
